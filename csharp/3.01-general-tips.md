# General Tips

> A structured approach to solving coding interview problems. Follow this **5-step framework** to stay organized and avoid jumping into code too early.

## The 5-Step Framework

1. **Clarify** -- Ask about edge cases, input constraints, and expected output format
2. **Brute force** -- State the naive solution and its time/space complexity
3. **Optimize** -- Identify bottlenecks; consider sorting, hashing, two pointers, or divide-and-conquer
4. **Code** -- Write clean, readable code with meaningful variable names
5. **Test** -- Walk through examples, edge cases (empty input, single element, duplicates)

## Common Complexity Targets

```
  n <= 10        ->  O(n!) or O(2^n)   backtracking, brute force
  n <= 20        ->  O(2^n)            bitmask DP
  n <= 500       ->  O(n^3)            Floyd-Warshall, 3 nested loops
  n <= 5,000     ->  O(n^2)            2D DP, pairwise comparison
  n <= 100,000   ->  O(n log n)        sorting, divide & conquer
  n <= 1,000,000 ->  O(n)              linear scan, hash map, sliding window
```

## C# Quick Tips

- Use `var` for local type inference to keep code concise
- `int.MaxValue` and `int.MinValue` for sentinel values
- `StringBuilder` for string concatenation in loops (not `+=`)
- Tuple deconstruction: `(a, b) = (b, a)` for swaps
- `Array.Sort(arr)` is in-place; `list.Sort()` for lists
- `Dictionary.TryGetValue` to avoid `KeyNotFoundException`
