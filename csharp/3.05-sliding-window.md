# Sliding Window

> The **sliding window** technique maintains a window `[left, right]` over a sequence, expanding right and shrinking left to satisfy a condition. Converts brute-force O(n^2) substring/subarray problems into **O(n)**. Two flavors: fixed-size window and variable-size window.

```
  "abcabcbb" - find longest substring without repeating chars

  Window slides:
    [a]bcabcbb        left=0, right=0  len=1
    [ab]cabcbb        left=0, right=1  len=2
    [abc]abcbb        left=0, right=2  len=3
    a[bca]bcbb        left=1, right=3  len=3  (duplicate 'a', move left)
    ab[cab]cbb        left=2, right=4  len=3  (duplicate 'b', move left)
    abc[abc]bb        left=3, right=5  len=3
    abca[bcb]b        left=4, right=6  len=3  (duplicate 'c', move left)
    abcab[cb]b        left=5, right=6  len=2  (duplicate 'b', move left)
    abcabc[b]b        left=6, right=6  len=1  ...
  Answer: 3 ("abc")
```

## Longest Substring Without Repeating Characters

Track the **last seen index** of each character. When a repeat is found inside the current window, jump `left` past the previous occurrence. Time: **O(n)**, Space: **O(1)** (fixed 256-char array).

```csharp
int LengthOfLongestSubstring(string s)
{
    // lastSeen[c] = last index where character c appeared
    var lastSeen = Enumerable.Repeat(-1, 256).ToArray();
    int best = 0, left = 0;

    for (int right = 0; right < s.Length; right++)
    {
        int ch = s[right];

        // If this char was seen inside current window, shrink window
        if (lastSeen[ch] >= left)
            left = lastSeen[ch] + 1;

        lastSeen[ch] = right;
        best = Math.Max(best, right - left + 1);
    }

    return best;
}
```
