# Max Profit Stock

> State-machine approach to stock trading problems. Track two states at each price:
> **profitNoStock** (best profit when not holding) and **profitHoldStock** (best profit when holding).
> At each price, decide whether to buy, sell, or do nothing. Time complexity: O(n) for all variants.

```
  State transitions at each price p:

    profitNoStock  = max(profitNoStock, profitHoldStock + p)   // sell or skip
    profitHoldStock = max(profitHoldStock, profitNoStock - p)   // buy or hold

  Example: prices = [7, 1, 5, 3, 6, 4]

    price=7: noStock=0, holdStock=-7
    price=1: noStock=0, holdStock=-1
    price=5: noStock=4, holdStock=-1
    price=3: noStock=4, holdStock=1
    price=6: noStock=7, holdStock=1
    price=4: noStock=7, holdStock=1
    Answer: 7
```

## Infinite Transactions

Unlimited buys and sells. Each sell can be followed by a new buy immediately.

```csharp
int MaxProfit(int[] prices)
{
    int profitNoStock = 0;
    int profitHoldStock = int.MinValue / 4;

    foreach (var price in prices)
    {
        int previousNoStock = profitNoStock;
        // Sell today or keep waiting
        profitNoStock = Math.Max(profitNoStock, profitHoldStock + price);
        // Buy today (using previous no-stock profit) or keep holding
        profitHoldStock = Math.Max(profitHoldStock, previousNoStock - price);
    }

    return profitNoStock;
}
```

## Single Transaction

At most one buy and one sell. The `profitHoldStock` transition uses `-price` instead of
`previousNoStock - price`, since we can only buy once (no prior profit to carry).

```csharp
int MaxProfit1(int[] prices)
{
    int profitNoStock = 0;
    int profitHoldStock = int.MinValue / 4;

    foreach (var price in prices)
    {
        // Sell today or keep waiting
        profitNoStock = Math.Max(profitNoStock, profitHoldStock + price);
        // Buy today (first buy, so cost is just -price) or keep holding
        profitHoldStock = Math.Max(profitHoldStock, -price);
    }

    return profitNoStock;
}
```

## K Transactions

At most k buy-sell pairs. Extend the two-state model to arrays of size k+1,
processing transactions from k down to 1 to avoid using updated values.

```csharp
int MaxProfitK(int k, int[] prices)
{
    var profitNoStock = new int[k + 1];
    var profitHoldStock = Enumerable.Repeat(int.MinValue / 4, k + 1).ToArray();

    foreach (var price in prices)
    {
        for (int i = k; i >= 1; i--)
        {
            // Sell on transaction i
            profitNoStock[i] = Math.Max(profitNoStock[i], profitHoldStock[i] + price);
            // Buy on transaction i (uses profit from completing transaction i-1)
            profitHoldStock[i] = Math.Max(profitHoldStock[i], profitNoStock[i - 1] - price);
        }
    }

    return profitNoStock[k];
}
```
