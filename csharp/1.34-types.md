# Types

> C# is statically typed -- generic collections require explicit type parameters.
> Common composite types for competitive programming include grids, adjacency lists, and nested dictionaries.

## Common Composite Type Declarations

```csharp
// 2D grid (list of lists)
var grid = new List<List<int>>();

// Adjacency list for a graph
var adjacencyList = new Dictionary<int, List<int>>();

// Frequency map (character -> count)
var frequency = new Dictionary<char, int>();

// Coordinate set
var visited = new HashSet<(int row, int col)>();

// Memoization cache
var memo = new Dictionary<(int, int), long>();
```

## Type Aliases (using Directives)

```csharp
// Shorten verbose types with using aliases (C# 12+)
using Grid = System.Collections.Generic.List<System.Collections.Generic.List<int>>;
using AdjList = System.Collections.Generic.Dictionary<int, System.Collections.Generic.List<int>>;

Grid board = new();
AdjList graph = new();
```

## var vs Explicit Type

```csharp
// var infers the type from the right-hand side
var numbers = new List<int>();           // inferred as List<int>
var lookup = new Dictionary<string, int>(); // inferred as Dictionary<string, int>

// Explicit type when clarity matters
List<List<int>> matrix = new();
```
