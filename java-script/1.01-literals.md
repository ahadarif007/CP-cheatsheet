# Literals

> JavaScript is dynamically typed -- variables don't declare a type, and any variable
> can hold any value. Primitives (`number`, `bigint`, `boolean`, `string`, `null`,
> `undefined`) are immutable value types. Objects, arrays, Maps, and Sets are reference types.

## Numeric Types

```js
const decimal = 255;              // base 10 integer
const binary  = 0b11111111;       // base 2 (prefix 0b) -> 255
const hex     = 0xff;             // base 16 (prefix 0x) -> 255
const big     = 12345678901234567890n; // BigInt for arbitrary precision
const float   = 1.23;             // 64-bit floating point (all numbers are doubles)
```

## Boolean, String, and Null / Undefined

```js
const isReady   = true;           // boolean
const letter    = 'a';            // string (no separate char type)
const greeting  = 'hello\n';     // string with escape sequence
const empty     = null;           // intentional absence of value
const notSet    = undefined;      // variable declared but not assigned
```

## Common Collection Initializers

```
Type                     Ordered?    Duplicates?    Lookup
-------------------------------------------------------------
Array []                 yes         yes            O(1) index
Map                      insertion   keys unique    O(1) key
Set                      insertion   no             O(1) has
Object {}                no*         keys unique    O(1) key
```

```js
const numbers = [1, 2, 3];                     // array -- dynamic, ordered
const lookup  = { key: 1 };                     // plain object -- string/symbol keys
const seen    = new Set(['a', 'b']);             // unique elements
const table   = new Map([['k', 1]]);            // key-value with any key type
```
