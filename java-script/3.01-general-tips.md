# General Tips

> A structured approach to solving coding problems in interviews and competitions.

## Five-Step Problem-Solving Framework

```
1. CLARIFY    - Restate the problem. Ask about edge cases, constraints, input size.
2. BRUTE FORCE - Think of the simplest correct solution first (even if O(n^2) or worse).
3. OPTIMIZE    - Identify bottlenecks. Apply patterns: two pointers, sliding window,
                 hash map, sorting, binary search, BFS/DFS, DP.
4. CODE        - Write clean, readable code. Use descriptive variable names.
5. TEST        - Walk through examples. Test edge cases: empty input, single element,
                 duplicates, negative numbers, large input.
```

## Common Complexity Targets by Input Size

```
n <= 10        O(n!)          brute-force permutations
n <= 20        O(2^n)         bitmask DP, subset enumeration
n <= 500       O(n^3)         triple loops, Floyd-Warshall
n <= 5,000     O(n^2)         double loops, simple DP
n <= 100,000   O(n log n)     sorting, binary search, segment tree
n <= 1,000,000 O(n)           hash map, two pointers, sliding window
```
