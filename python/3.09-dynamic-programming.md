## Dynamic Programming

DP solves problems by breaking them into overlapping subproblems, solving each once, and storing results. Key indicators: "optimal" (min/max), overlapping subproblems, choice at each step.

[DP patterns](https://leetcode.com/discuss/general-discussion/458695/Dynamic-Programming-Patterns)

**Coin Change** - minimum coins to make an amount (unbounded knapsack):

```python
def coinChange(self, coins: List[int], amount: int) -> int:
  MAX = float('inf')
  dp = [MAX] * (amount + 1)   # dp[i] = minimum coins needed to make amount i
  dp[0] = 0                    # base case: 0 coins needed for amount 0

  for c in coins:              # try each coin denomination
    for a in range(c, amount+1):   # for each amount from c to target
      # Choice: either don't use coin c (keep dp[a])
      # OR use coin c: 1 + best way to make (amount - coin_value)
      dp[a] = min(dp[a], dp[a-c] + 1)

  return dp[amount] if dp[amount] != MAX else -1  # -1 if impossible
```

**Longest Common Subsequence (LCS)** - classic 2D DP:

```python
def longestCommonSubsequence(self, text1: str, text2: str) -> int:
    Y = len(text2) + 1     # +1 for empty string base case
    X = len(text1) + 1
    # dp[i][j] = length of LCS of text1[:i] and text2[:j]
    dp = [[0] * Y for _ in range(X)]  # all zeros initially

    for i, c in enumerate(text1):
        for j, d in enumerate(text2):
            if c == d:
                # Characters match! LCS extends by 1 from diagonal
                dp[i+1][j+1] = 1 + dp[i][j]
            else:
                # Characters don't match: take the better of
                # excluding current char from either string
                dp[i+1][j+1] = max(dp[i][j+1], dp[i+1][j])

    return dp[-1][-1]   # bottom-right corner = final answer

# Example: text1="abcde", text2="ace"
# The LCS is "ace" (length 3)
# dp grid shows how subproblems build up to the answer
```
