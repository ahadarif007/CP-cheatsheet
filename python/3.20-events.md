## Events

The events (sweep line) pattern flattens intervals into +1/-1 events, sorts by time, then sweeps through. This is powerful for: employee free time, meeting rooms needed, peak concurrent users, etc.

The idea: instead of thinking about intervals as a whole, think about what happens at each point in time (someone starts/someone ends).

```python
def employeeFreeTime(self, schedule: '[[Interval]]') -> '[Interval]':
    events = []
    for e in schedule:
        for m in e:
            events.append((m.start, 1))    # +1 event: a meeting starts (one more person busy)
            events.append((m.end, -1))     # -1 event: a meeting ends (one fewer person busy)
    events.sort()                          # sort by time (ties broken by value: end before start)
    itv = []
    prev = None
    bal = 0                                # running count of active meetings
    for t, c in events:
        if bal == 0 and prev is not None and t != prev:
            itv.append(Interval(prev, t))  # gap where nobody is busy = free time!
        bal += c                           # update active count
        prev = t
    return itv
```
