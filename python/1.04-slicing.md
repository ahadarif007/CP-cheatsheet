## Slicing

Slicing is one of Python's most powerful features. It lets you extract portions of lists/strings using the syntax `[start:stop:step]`. The key insight: slice positions are BETWEEN elements (like fence posts), while indices point AT elements.

```python
                +---+---+---+---+---+---+
                | P | y | t | h | o | n |
                +---+---+---+---+---+---+
Slice position: 0   1   2   3   4   5   6    # positions between elements
Index position:   0   1   2   3   4   5      # positions of elements

p = ['P','y','t','h','o','n']

# INDEXING vs SLICING - different return types!
p[0]           # 'P'              - indexing returns a single ITEM
p[0:1]         # ['P']            - slicing returns a LIST (even with one element)

# BASIC SLICING: [start:stop] - start is included, stop is EXCLUDED
p[0:5]         # ['P','y','t','h','o']  - elements at index 0,1,2,3,4 (not 5!)
alpha[slice(2,4)]  # equivalent to p[2:4] - slice() object form (rarely used directly)

# SLICE ASSIGNMENT - you can replace a slice with a different-sized list!
p[2:4] = ['t','r']           # Replace indices 2,3: ['P','y','t','r','o','n']
p[2:4] = ['s','p','a','m']  # Replace 2 elements with 4: ['P','y','s','p','a','m','o','n']
p[4:4] = ['x','y']          # Insert at position 4 (empty slice = insert): ['P','y','t','h','x','y','o','n']

# STEP parameter: [start:stop:step]
p[0:5:2]      # ['P', 't', 'o']  - every 2nd element from index 0 to 4
p[5:0:-1]     # ['n', 'o', 'h', 't', 'y']  - backward from index 5 to 1 (not 0!)
p[::-1]       # reverses entire list (very common pattern!)
```

**Negative indexing** - access elements from the end:

```python
a = [10, 20, 30, 40, 50]
a[-1]              # 50 (last element)
a[-2]              # 40 (second to last)
a[-3:]             # [30, 40, 50] (last 3 elements)
a[:-2]             # [10, 20, 30] (everything except last 2)
```

**Common CP slicing patterns:**

```python
# Rotate array right by k positions
def rotate(a, k):
    k %= len(a)                    # handle k > len(a)
    return a[-k:] + a[:-k]        # last k elements + first n-k elements

rotate([1,2,3,4,5], 2)            # [4, 5, 1, 2, 3]
```

**Get all combinations missing one member** - useful for "leave one out" problems:

```python
numList = [1,2,3,4]
for i in range(len(numList)):
    # numList[0:i] = everything BEFORE i
    # numList[i+1:len(numList)] = everything AFTER i
    newList = numList[0:i] + numList[i+1:len(numList)]
    print(newList)  # [2,3,4], [1,3,4], [1,2,4], [1,2,3]
```
