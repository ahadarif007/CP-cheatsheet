## Cyclic Sort

Cyclic sort places each number at its "correct" index in O(n) time using O(1) space. Works when elements are in range [1, n] or [0, n-1]. The key insight: keep swapping current element to its correct position until current position has the right element.

```
  nums = [3, 1, 5, 4, 2]       goal: place n at index n-1

  i=0: nums[0]=3, should be at index 2 → swap → [5, 1, 3, 4, 2]
  i=0: nums[0]=5, should be at index 4 → swap → [2, 1, 3, 4, 5]
  i=0: nums[0]=2, should be at index 1 → swap → [1, 2, 3, 4, 5]
  i=0: nums[0]=1, correct! → move to i=1
  i=1: nums[1]=2, correct! → move to i=2
  ... all correct!

  Result: [1, 2, 3, 4, 5]
```

```python
def cyclic_sort(nums):
    i = 0
    while i < len(nums):
        j = nums[i] - 1                          # correct index for nums[i]
        if nums[i] != nums[j]:                   # not in correct position
            nums[i], nums[j] = nums[j], nums[i]  # swap it there
            # DON'T increment i — swapped-in element might also be wrong
        else:
            i += 1                                # correct (or duplicate), move on
    return nums
```

**Find missing number** - after cyclic sort, scan for mismatches:

```python
def findMissingNumber(nums):
    # For range [0, n]: place num at index num
    i, n = 0, len(nums)
    while i < n:
        j = nums[i]
        if nums[i] < n and nums[i] != nums[j]:
            nums[i], nums[j] = nums[j], nums[i]
        else:
            i += 1
    for i in range(n):
        if nums[i] != i:
            return i           # this index is missing its number
    return n
```
