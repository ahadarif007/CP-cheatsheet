## Literals

Literals are fixed values written directly in code. Python infers the type from syntax.

```python
# === NUMBERS ===
255, 0b11111111, 0o377, 0xff  # Integer 255 in decimal, binary, octal, hex
1_000_000                      # Underscores for readability (== 1000000)
123.0, 1.23e4                  # Float; 1.23e4 == 12300.0
float('inf'), float('-inf')    # Infinity (useful for min/max initialization)
7 + 5j                         # Complex (rarely needed in CP)

# === STRINGS ===
'a', '\141', '\x61'           # Character 'a': literal, octal escape, hex escape
'\n', '\\', '\t'              # Newline, backslash, tab
"hello" + "world"             # Concatenation: "helloworld"
"ab" * 3                      # Repetition: "ababab"
f"{x} + {y} = {x+y}"         # f-string (Python 3.6+): inline expressions

# === BOOLEANS ===
True, False                   # True == 1, False == 0 (usable in arithmetic)
                               # sum(condition for x in arr) counts truthy items

# === DATA STRUCTURES ===
[1, 2, 3]                    # List: ordered, mutable, O(1) index/append, O(n) insert/search
(2, 4, 6)                    # Tuple: ordered, immutable, hashable (usable as dict key/set element)
{'a': 1, 'b': 2}             # Dict: O(1) avg lookup/insert by key, ordered (3.7+)
{'a', 'b', 'c'}              # Set: O(1) avg lookup/add, no duplicates, unordered
None                          # Null value

# === CP-USEFUL PATTERNS ===
MOD = 10**9 + 7               # Common modulo constant
INF = float('inf')            # For Dijkstra, DP initialization
dx = [-1, 0, 1, 0]           # Direction arrays for grid BFS/DFS
dy = [0, 1, 0, -1]
DIRS = [(-1,0),(0,1),(1,0),(0,-1)]  # Alternative: list of tuples
```

### Quick Reference: When to Use What

| Structure | Use When | Key Advantage |
|-----------|----------|---------------|
| `list`    | Ordered collection, need index access | O(1) append, flexible |
| `tuple`   | Need hashable sequence (dict key, set element) | Immutable, less memory |
| `dict`    | Key→value mapping, counting, memoization | O(1) lookup |
| `set`     | Membership checks, deduplication | O(1) `in` operator |

### Common Pitfalls

```python
# Mutable default: both point to SAME list
a = b = []          # WRONG for independent lists
a, b = [], []       # CORRECT

# 2D array initialization
grid = [[0]*n for _ in range(m)]   # CORRECT
grid = [[0]*n] * m                  # WRONG: all rows share same reference

# Integer limits: Python has arbitrary precision (no overflow!)
2**1000                             # Works fine, no need for BigInteger
```
