## Modulo

The modulo operator `%` gives the remainder after division. In Python, the result always has the same sign as the divisor (different from C/Java!). `//` is integer division (floor division).

```python
for n in range(-8, 8):
    print(n, n//4, n%4)

#  n   n//4  n%4
# -8   -2    0     # -8 = -2*4 + 0
# -7   -2    1     # -7 = -2*4 + 1 (NOT -3*4 - 1 like in C!)
# -6   -2    2     # Python: remainder is always non-negative when divisor is positive
# -5   -2    3
# -4   -1    0
# -3   -1    1
# -2   -1    2
# -1   -1    3     # -1 % 4 = 3 in Python (but -1 in C/Java!)
#  0    0    0
#  1    0    1
#  2    0    2
#  3    0    3
#  4    1    0
#  5    1    1
#  6    1    2
#  7    1    3
```

**`divmod(a, b)`** - returns both quotient and remainder in one call:

```python
divmod(7, 3)    # (2, 1)  ->  7 = 2*3 + 1
divmod(-7, 3)   # (-3, 2) ->  -7 = -3*3 + 2 (Python floor division)

# Useful for converting units:
minutes, seconds = divmod(total_seconds, 60)
hours, minutes = divmod(total_minutes, 60)
```

**Modular arithmetic properties** - useful for keeping numbers small in CP:

```python
(a + b) % m == ((a % m) + (b % m)) % m
(a * b) % m == ((a % m) * (b % m)) % m
# Subtraction: add m to avoid negative results
(a - b) % m == ((a % m) - (b % m) + m) % m
```

**Common CP pattern** - wrap-around indexing with modulo:

```python
# Circular array: next element after the last wraps to the first
arr = [10, 20, 30, 40]
for i in range(len(arr)):
    next_i = (i + 1) % len(arr)   # 0→1, 1→2, 2→3, 3→0 (wraps!)
    print(arr[i], "→", arr[next_i])
```
