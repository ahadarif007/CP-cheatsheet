## Heapq

A heap (priority queue) is a binary tree where the parent is always smaller than its children (min-heap). This means the smallest element is always at the root, giving us O(1) access to the minimum and O(log n) insert/remove.

```
      1         <- smallest element always at top
     / \
    2   3
   / \ / \
  5  6 8  7
```

[Priority Queue](https://realpython.com/python-heapq-module/#data-structures-heaps-and-priority-queues)

1. Implemented as complete binary tree, which has all levels full except deepest
1. In a min-heap, every parent node is smaller than its children
1. Python's `heapq` is a MIN-heap (smallest first). For max-heap, negate values.

```python
def maximumProduct(self, nums: List[int]) -> int:
  l = heapq.nlargest(3, nums)    # get 3 largest numbers efficiently
  s = heapq.nsmallest(3, nums)   # get 3 smallest numbers
  # Why check both? Example: [-10, -9, 1, 2, 3]
  # 3*2*1 = 6, BUT (-10)*(-9)*3 = 270 (two negatives make positive!)
  return max(l[0]*l[1]*l[2], s[0]*s[1]*l[0])
```

**Max-heap trick** - Python only has min-heap. To get max-heap behavior, store NEGATIVE values. When you pop the "smallest" negative, it's actually the largest original value:

```python
def kClosest(self, points: List[List[int]], K: int) -> List[List[int]]:
    heap = []                     # will store (-distance, point) tuples
    for p in points:
        distance = sqrt(p[0]*p[0] + p[1]*p[1])  # Euclidean distance from origin
        heapq.heappush(heap, (-distance, p))     # NEGATE distance for max-heap
        if len(heap) > K:
            heapq.heappop(heap)   # remove the LARGEST distance (smallest negative)
    # Heap now contains K closest points
    return ([h[1] for h in heap]) # extract just the points (index 1 of each tuple)
```

**nsmallest/nlargest with key function** - cleaner than manual heap operations:

```python
def kClosest(self, points: List[List[int]], K: int) -> List[List[int]]:
    # lambda defines "distance" without actually computing sqrt (not needed for comparison)
    return heapq.nsmallest(K, points, lambda x: x[0]*x[0] + x[1]*x[1])
```

```python
def topKFrequent(self, nums: List[int], k: int) -> List[int]:
    count = Counter(nums)                    # count frequency of each number
    return heapq.nlargest(k, count, count.get)  # k numbers with highest frequency
    # count.get is a function that returns the count for each key
```

**Sorting tuples in heap** - when you push tuples, heap compares by first element, then second for ties:

```python
def topKFrequent(self, words: List[str], k: int) -> List[str]:
    freq = Counter(words)
    # nsmallest with key=(-frequency, word):
    # - most frequent first (negate freq for "smallest" = highest freq)
    # - alphabetical order for ties (word compared normally)
    return heapq.nsmallest(k, freq.keys(), lambda x: (-freq[x], x))
```
