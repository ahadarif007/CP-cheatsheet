## Sort

Python has two ways to sort: `sorted()` (returns a NEW sorted list) and `.sort()` (sorts in-place, returns None). Both use Timsort algorithm which is O(n log n).

The `key` parameter lets you customize what to sort by - it takes a function that transforms each element into a "comparison value".

```python
# sorted() returns a new list, original unchanged
print(sorted(['Ford', 'BMW', 'Volvo']))  # ['BMW', 'Ford', 'Volvo'] - alphabetical

# Sorting with transformation - sort by squared values
nums = [-4,-1,0,3,10]
print(sorted(n*n for n in nums))  # [0, 1, 9, 16, 100]
```

```python
cars = ['Ford', 'BMW', 'Volvo']
cars.sort()                        # modifies cars in-place, returns None!
cars.sort(key=lambda x: len(x))   # sort by string length: ['BMW', 'Ford', 'Volvo']
print(sorted(cars, key=lambda x: len(x)))  # same result but returns new list
```

**Sort dictionary keys by their values** - when you iterate over a dict with `sorted()`, you get sorted keys. Use `key` to sort them by something else:

```python
meh = {'a':3, 'b':0, 'c':2, 'd':-1}
# sorted(meh) iterates over KEYS, but key=lambda sorts by VALUES
print(sorted(meh, key=lambda x: meh[x]))  # ['d', 'b', 'c', 'a']
# Explanation: d=-1 < b=0 < c=2 < a=3

# Works even when values are lists (compares element by element)
meh = {'a':[0,3,'a'], 'b':[-2,-3,'b'], 'c':[2,3,'c'], 'd':[-2,-2,'d']}
print(sorted(meh, key=lambda x: meh[x]))  # ['b', 'd', 'a', 'c']
# b=[-2,-3] < d=[-2,-2] (first elements equal, compare second) < a=[0,3] < c=[2,3]
```

```python
# Timsort is optimized for partially sorted data - merging two sorted lists is fast
def merge_sorted_lists(arr1, arr2):
    return sorted(arr1 + arr2)
```

**Keep original indices after sorting** - useful when you need both sorted values and their original positions:

```python
# enumerate(nums) gives (index, value) pairs
# sorted by value (x[1]), then unzip with zip(*...) to get separate index and value lists
self.idx, self.vals = zip(*sorted([(i,v) for i,v in enumerate(nums)], key=lambda x:x[1]))
```

**Multi-key sorting with tuples** - Python compares tuples element by element. Use negative values to reverse specific keys:

```python
a = [(5,10), (2,20), (2,3), (0,100)]

# Sort by 2nd element first, then 1st element (both ascending)
test = sorted(a, key=lambda x: (x[1], x[0]))
print(test)  # [(2, 3), (5, 10), (2, 20), (0, 100)]

# Sort by 2nd element DESCENDING (negate it), then 1st element ascending
test = sorted(a, key=lambda x: (-x[1], x[0]))
print(test)  # [(0, 100), (2, 20), (5, 10), (2, 3)]
# Trick: negate a number to reverse its sort order in that dimension
```

**Custom comparator with cmp_to_key** - when sort logic can't be expressed as a simple key. Returns negative/zero/positive like C's strcmp:

```python
from functools import cmp_to_key

# Largest Number problem: arrange nums to form the largest number
# Compare "3" vs "30": "330" > "303", so 3 should come first
nums = [3, 30, 34, 5, 9]
def compare(a, b):
    if a + b > b + a: return -1    # a should come first (descending)
    elif a + b < b + a: return 1
    return 0

nums = [str(n) for n in nums]
nums.sort(key=cmp_to_key(compare))
result = ''.join(nums)             # "9534330"
```

**Stable sort** - Python's sort is stable: equal elements keep their original order. This lets you sort by multiple criteria in separate passes (sort by secondary key first, then primary):

```python
# Sort by score descending, then by name ascending (for ties)
data = [("Alice", 90), ("Bob", 90), ("Charlie", 85)]
data.sort(key=lambda x: x[0])        # sort by name first (secondary)
data.sort(key=lambda x: -x[1])       # then by score descending (primary)
# [("Alice", 90), ("Bob", 90), ("Charlie", 85)]
```

**Sort and print dict values by key:**

```python
ans = {-1: [(10, 1), (3, 3)], 0: [(0, 0), (2, 2), (7, 4)], -3: [(8, 5)]}
for key, value in sorted(ans.items()):  # .items() gives (key, value) pairs
    print(value)
# [(8, 5)]              (key -3)
# [(10, 1), (3, 3)]    (key -1)
# [(0, 0), (2, 2), (7, 4)]  (key 0)

# What sorted() does to different dict views:
sorted(ans)          # [-3, -1, 0]  (sorts keys only)
sorted(ans.values()) # sorts the value lists
sorted(ans.items())  # sorts (key, value) tuples by key

# Shorthand to get values in key-sorted order:
[ans[i] for i in sorted(ans)]
```
