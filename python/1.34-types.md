## Types

Python type hints help document what types functions expect. They don't enforce types at runtime but help with readability and IDE support.

**Common type hints for CP:**

```python
from typing import List, Set, Dict, Tuple, Optional

def two_sum(nums: List[int], target: int) -> List[int]:
    ...

def build_graph(edges: List[List[int]]) -> Dict[int, List[int]]:
    ...

def find_node(root: Optional[TreeNode]) -> Optional[int]:
    ...
```

**Quick reference:**

```
List[int]              list of integers
Dict[str, int]         dict with str keys, int values
Set[int]               set of integers
Tuple[int, str]        tuple of exactly (int, str)
Tuple[int, ...]        tuple of any number of ints
Optional[int]          int or None  (same as Union[int, None])
```

**Python 3.9+ built-in generics** (no import needed):

```python
def solve(nums: list[int], graph: dict[int, list[int]]) -> tuple[int, int]:
    ...
# lowercase list, dict, tuple, set work directly — no typing import needed
```
