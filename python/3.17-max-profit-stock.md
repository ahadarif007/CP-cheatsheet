## Max Profit Stock

Stock problems follow a state machine pattern: at each price, you're either holding stock (state t1) or not holding (state t0). Transitions: buy (t0→t1) and sell (t1→t0).

**Infinite transactions** - [base formula](https://leetcode.com/problems/best-time-to-buy-and-sell-stock-with-cooldown/discuss/75924/Most-consistent-ways-of-dealing-with-the-series-of-stock-problems):

```python
def maxProfit(self, prices: List[int]) -> int:
    # t0 = max profit when NOT holding stock
    # t1 = max profit when HOLDING stock
    t0, t1 = 0, float('-inf')     # initially: no profit, haven't bought yet
    for p in prices:
        t0old = t0                # save before updating (both transitions use old values)
        t0 = max(t0, t1 + p)     # sell: profit = was holding + sell price
        t1 = max(t1, t0old - p)  # buy: profit = was not holding - buy price
    return t0
```

**Single transaction** - same pattern but t1 can only buy from 0 profit (only one buy allowed):

```python
def maxProfit(self, prices: List[int]) -> int:
    t0, t1 = 0, float('-inf')
    for p in prices:
        t0 = max(t0, t1 + p)     # sell (best sell so far)
        t1 = max(t1, -p)         # buy: profit = 0 - price (only one buy, starts from 0)
    return t0
```

**K transactions:**

```python
t0 = [0] * (k+1)                 # t0[i] = max profit with i transactions, not holding
t1 = [float(-inf)] * (k+1)       # t1[i] = max profit with i transactions, holding
for p in prices:
    for i in range(k, 0, -1):     # process from k down to 1
        t0[i] = max(t0[i], t1[i] + p)      # sell using transaction i
        t1[i] = max(t1[i], t0[i-1] - p)    # buy using transaction i (from i-1 state)
return t0[k]
```
