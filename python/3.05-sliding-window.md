## Sliding Window

A technique for problems involving contiguous subarrays/substrings. Instead of checking every possible subarray (O(n^2)), we maintain a "window" that slides across the array (O(n)).

**Pattern:**
1. Expand window to the right (add elements)
2. When window violates constraint, shrink from the left (remove elements)
3. Track the best valid window seen

### Typical Problem Clues:

1. Get min/max/number of satisfied sub arrays
1. Return length of the subarray with max sum/product
1. Return max/min length/number of subarrays whose sum/product equals K

Can require [2 or 3 pointers to solve](https://medium.com/algorithms-and-leetcode/magic-solution-to-leetcode-problems-sliding-window-algorithm-891e3d60bf89)

**General template:**

```python
def slidingWindowTemplate(self, s: str):
    rtn = []             # result (varies by problem)
    hm = {}              # hashmap to track window contents
    cnt = collections.Counter(s)  # counter for condition checking
    l = r = 0           # left and right pointers of window

    for r, c in enumerate(s):     # expand window to the right

        if c in hm:
            l = max(hm[c]+1, l)  # move left pointer past duplicate
                                  # max() ensures we never move l backward!

        hm[c] = r               # record latest position of character

        while cnt == 0:          # while window is invalid (condition varies)
            cnt[c] += 1         # modify counter to make window valid again

        # Window is valid here - update answer
        rtn = max(rtn, r-l+1)   # window size = right - left + 1

    return rtn
```

**Concrete example: max fruits in 2 baskets** (at most 2 distinct fruit types):

```python
def fruits_into_baskets(fruits):
  maxCount, j = 0, 0      # maxCount = best answer, j = left pointer
  ht = {}                  # tracks count of each fruit type in window

  for i, c in enumerate(fruits):   # i = right pointer, c = current fruit
    if c in ht:
      ht[c] += 1          # fruit already in basket, add one more
    else:
      ht[c] = 1           # new fruit type

    if len(ht) <= 2:      # window valid: at most 2 fruit types
      maxCount = max(maxCount, i-j+1)  # update best window size
    else:                  # window invalid: more than 2 types
      jc = fruits[j]      # fruit at left edge
      ht[jc] -= 1         # remove one from left
      if ht[jc] <= 0:
        del ht[jc]        # if count reaches 0, remove type entirely
      j += 1              # shrink window from left

  return maxCount
```
