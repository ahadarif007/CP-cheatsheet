# Dict (HashMap)

> `HashMap` stores **key-value pairs** with O(1) average lookup.
> Keys must be unique — putting a duplicate key overwrites the old value.
> Iteration order is **not guaranteed** (use `LinkedHashMap` for insertion order).

## Common Operations

```java
Map<String, Integer> scores = new HashMap<>();

// Insert / update
scores.put("alice", 95);
scores.put("bob", 87);

// Get value with a default if key is missing
int aliceScore = scores.getOrDefault("alice", 0);   // 95
int unknownScore = scores.getOrDefault("charlie", 0); // 0

// Iterate over keys only
for (String name : scores.keySet()) {
    System.out.println(name);
}

// Iterate over key-value pairs
for (var entry : scores.entrySet()) {
    String name = entry.getKey();
    int score = entry.getValue();
}

// Create key with default value if absent (useful for adjacency lists)
scores.computeIfAbsent("charlie", key -> 0);
// charlie=0 is now in the map

// Remove a key
scores.remove("bob");
```

```
HashMap internals (simplified):
┌──────────┐
│ Buckets  │
├──────────┤     ┌─────────────────┐
│  [0]     │────→│ "alice" = 95    │
├──────────┤     └─────────────────┘
│  [1]     │
├──────────┤     ┌─────────────────┐
│  [2]     │────→│ "bob" = 87      │
├──────────┤     └─────────────────┘
│  ...     │
└──────────┘

put/get/remove are O(1) average.
Keys are hashed → bucket index → stored in that bucket.
```
