# Literals

> Java is statically typed — every variable must declare its type.
> Primitive types (`int`, `long`, `double`, `boolean`, `char`) are stored on the stack.
> Wrapper/reference types (`Integer`, `String`, `List`, etc.) are stored on the heap.

## Primitive Types

```java
// Integer literals in different bases
int decimal    = 255;           // base 10
int binary     = 0b11111111;    // base 2 (prefix 0b)
int hex        = 0xFF;          // base 16 (prefix 0x)

// Long requires 'L' suffix (for values > 2^31 - 1)
long bigNumber = 1234567890123L;

// Floating point
double pi      = 3.14;

// Boolean — only true or false
boolean isReady = true;

// Single character (uses single quotes)
char letter    = 'a';

// Null reference (only for objects, not primitives)
Object nothing = null;
```

## Common Collection Declarations

```java
// Array — fixed size, declared with {}
int[] numbers = {1, 2, 3};

// ArrayList — dynamic size, generic type required
List<Integer> list = new ArrayList<>();

// HashMap — key-value pairs
Map<String, Integer> map = new HashMap<>();

// HashSet — unique elements, no duplicates
Set<Character> charSet = new HashSet<>();
```

```
Primitive vs Wrapper Types:
┌──────────┬───────────┬──────────────────────────┐
│ Primitive│  Wrapper  │  Notes                   │
├──────────┼───────────┼──────────────────────────┤
│ int      │ Integer   │ Collections need wrappers │
│ long     │ Long      │ Autoboxing converts auto  │
│ double   │ Double    │ int → Integer             │
│ boolean  │ Boolean   │ Integer → int (unboxing)  │
│ char     │ Character │                           │
└──────────┴───────────┴──────────────────────────┘
```
