# Binary Search

Binary search finds a target in **O(log n)** by halving the search space each step.
Key insight: use `mid = left + (right - left) / 2` to avoid integer overflow.

```
Search space: [left ... mid ... right]

If condition(mid) is true  -> shrink right half:  right = mid
If condition(mid) is false -> shrink left half:   left = mid + 1

Loop ends when left == right -> answer found
```

## First Bad Version (Find Leftmost True)

Given versions `[1, 2, ..., n]`, find the first bad one. `isBad(m)` returns true if version `m` is bad.

```
Version:  1  2  3  4  5  6  7
isBad:    F  F  F  T  T  T  T
                   ^
                   first bad = 4
```

```java
int firstBadVersion(int totalVersions) {
    int left = 1, right = totalVersions;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (isBad(mid)) {
            right = mid;       // mid might be the answer, keep it
        } else {
            left = mid + 1;    // mid is good, skip it
        }
    }
    return left; // left == right, converged on the answer
}
```

## Integer Square Root

Find the largest integer whose square is <= x.

```
x = 8
Candidates: 1  2  3  4  5 ...
Squares:    1  4  9  16 25 ...
                   ^
                   9 > 8, so answer = 2
```

```java
int mySqrt(int x) {
    if (x <= 1) return x;

    int left = 1, right = x;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if ((long) mid * mid > x) {
            right = mid;       // mid is too large
        } else {
            left = mid + 1;    // mid might be answer or too small
        }
    }
    return left - 1; // left overshoots by 1, so subtract
}
```
