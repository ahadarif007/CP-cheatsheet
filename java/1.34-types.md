# Types (Complex Type Declarations)

> Java generics require explicit type parameters for nested collections.
> These are the most common complex types used in competitive programming.

```java
// 2D list — used for grids, matrix operations
List<List<Integer>> grid = new ArrayList<>();

// Adjacency list — maps node to list of neighbors
Map<Integer, List<Integer>> adjacencyList = new HashMap<>();

// Frequency map of strings
Map<String, Integer> wordCount = new HashMap<>();

// Group anagrams: sorted string → list of original strings
Map<String, List<String>> anagramGroups = new HashMap<>();

// Graph with weighted edges: node → list of [neighbor, weight]
Map<Integer, List<int[]>> weightedGraph = new HashMap<>();
```

> **Tip**: Use `computeIfAbsent` to initialize nested collections:
> ```java
> // Add neighbor to adjacency list, creating the list if needed
> adjacencyList.computeIfAbsent(node, k -> new ArrayList<>()).add(neighbor);
> ```
