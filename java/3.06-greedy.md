# Greedy

Greedy algorithms make the **locally optimal choice** at each step, hoping it leads to a globally optimal solution.

## Increasing Triplet Subsequence

Find if there exist indices `i < j < k` such that `nums[i] < nums[j] < nums[k]`.

**Strategy:** Track the smallest and second-smallest values seen so far.
If any value is greater than both, we found a triplet.

```
  nums: [2, 1, 5, 0, 4, 6]

  Step by step:
    2  -> smallest=2, secondSmallest=MAX
    1  -> smallest=1, secondSmallest=MAX     (update smallest)
    5  -> smallest=1, secondSmallest=5       (update secondSmallest)
    0  -> smallest=0, secondSmallest=5       (update smallest)
    4  -> smallest=0, secondSmallest=4       (update secondSmallest)
    6  -> 6 > secondSmallest(4) -> FOUND!    (triplet exists)
```

Note: `smallest` may update past `secondSmallest`, but that's fine - `secondSmallest` still implies a smaller value existed before it.

```java
boolean increasingTriplet(int[] nums) {
    int smallest = Integer.MAX_VALUE;
    int secondSmallest = Integer.MAX_VALUE;

    for (int num : nums) {
        if (num <= smallest) {
            smallest = num;             // New smallest value
        } else if (num <= secondSmallest) {
            secondSmallest = num;       // New second smallest
        } else {
            return true;                // Found: smallest < secondSmallest < num
        }
    }
    return false;
}
```
