# General Tips for Coding Interviews

Follow the **CUBOC** framework for every problem:

```
1. CLARIFY     -> Ask about edge cases, constraints, input size
2. BRUTE FORCE -> Start with the simplest working solution
3. OPTIMIZE    -> Identify bottlenecks, apply patterns
4. CODE        -> Write clean, modular code
5. TEST        -> Walk through examples, edge cases, dry run
```

## Key Reminders

- **Always clarify** input constraints (size, range, sorted?, duplicates?)
- **State the brute force** first, even if you won't code it - shows structured thinking
- **Analyze time & space** complexity before and after optimizing
- **Talk through your thought process** - interviewers evaluate how you think
- **Test with edge cases**: empty input, single element, all duplicates, very large input
